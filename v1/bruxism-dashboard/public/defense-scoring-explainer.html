Defense Cascade Explainer

This page visualizes the exact scenarios used in the defense-scoring tests. It runs the same computeDefenseScores implementation used by the dashboard and compares it to the prior additive model to show why the updated mechanics better match biology.

Mechanics under test: source-aware cascade + noisy-OR merge
Decay per hop: 0.8
Goal: no single-source amplification through split/rejoin topology

Biological Basis

  • A defense signal from one upstream cause should not grow just because the causal graph branches and re-merges.
  • Converging independent causes can increase protection, but with diminishing returns rather than linear addition.
  • Feedback/protective annotation edges are not part of the forward defense-cascade transport in this model.

Model Choices

  • Per-source max-path rule: each source contributes its strongest path to each node (prevents topology-induced double counting).
  • Noisy-OR merge: independent source contributions combine as 1 - Î (1 - p).
  • Hop decay: each forward hop multiplies by CASCADE_DECAY to represent attenuation across causal distance.

Verification guidance: For each scenario, compare Current model and Legacy additive model node scores. In split/rejoin cases, the current model should stay bounded by upstream source strength, while the legacy model inflates.
